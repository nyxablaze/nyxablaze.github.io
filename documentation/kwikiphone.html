KwikiPhone Rentals - Web app project

Details:

This project was made for CITS1402's 2023 project. The code is currently private - as the project is still ongoing and the code for my web app contains answers to parts of the project. Sorry guys :]
The web app is intended to be used on a kiosk/clerk machine running on a local server at the KwikiPhone Rentals facility. Data integrity checks are performed in the background for crucial functions such as IMEI and customer ID, so that it is difficult to have incorrect information.
All in all, it's relatively simple Python code. Most every page's backend workings are about the same, just tweaked to the specific requirements of eaech table in the SQL database.

Challenges


I faced some issues with getting foreign keys to work, even after including the execution of PRAGMA foreign_keys = ON; in the python code. In the end, I just used extra triggers, which solved the problem.
I had to add error messages, as originally it would just return you to the same apge.
Cybersecurity: there is no SQL Injection protection - the assumption is this will not be public and the employees will not be malicious.

Improvements possible


The styling could be nicer, but I don't have time to tweak it anymore.
The password protection feature is clunky - it only logs in everytime the browser is refreshed, there are no cookies used. This could be made less clunky.

Worth mentioning:


It is important to note the CustomerID generation procedure - the program has a file, calling idlist.txt, and everytime a new customer is entered, the following procedure occurs:
idlist.txt is opened. A random number between 1000000 and 9999999 is selected by the program. idlist.txt is then read and searched for this randomly generated integer. If it exists, the program repeats this process. If it doesn't exist in idlist.txt, the integer is used as the new customer's ID number, and it is appeneded to idlist.txt in order to log that the ID is now in use.
