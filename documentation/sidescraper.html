HOW IT WORKS

Basically, this program, nicknamed sidescraper, is a HTML frontend (what you see) with a python based backend (where all the work is done, a.k.a an API). The frontend contains the forms with make, model, year, variant and bodytype.
When you input your data and hit 'submit', it gets sent to the API. This API then uses these data fields to literally just construct the URL of the carsales page for the car you're looking for. e.g.:
• model: honda
• make: civic
• year: 2018
• variant: vti-l
• bodytype: hatchback
constructs the url: https://www.carsales.com.au/honda/civic/price/2018/vti-l-
what you may notice though is that 1. the website has the 'price' in the URL, the URL has a '-' on the end of the variant, and 3 the URL isn't complete. there are two types of 2018 vti-l's, the sedan and the hatchabck, this is where the bodytype parameter is used.
using the constructed URL so far, the API looks for all urls leading off of this one, and in this case there is two:
• https://www.carsales.com.au/honda/civic/price/2018/vti-l-479753/
and
• https://www.carsales.com.au/honda/civic/price/2018/vti-l-479759/
these differ by two numbers and don't explictly state the body type. However, you can see why the dash is added after the variant, because otherwise its ambiguous if the user was referring to the vti-l or vti-lx, which has it's own set of URLs.
the next step is for the API to open this URL, and look through it. If it doesn't find the right bodytype, it loops to the next URL. When the bodytype is found, the loop breaks, and the URL that broke the loop is used. In this case, the bodytype hatchback found in the site https://www.carsales.com.au/honda/civic/price/2018/vti-l-479753/, under "Body: Hatch". my code takes care of the difference between Hatchback and Hatch.
Next, the successful URL is downloaded to the backend. The locally stored URL is then opened by the API, and the price data and image are extracted from this, and displayed back to you on the frontend.
That's it

USAGE TIPS

• Insert the right spelling exactly for all fields
• For Variant: Include dashes in cars where it's part of the name ('v-ti' not 'vti', 'mx-5' not 'mx5'). Spaces will be converted to dashes, but usually the user doesn't need to worry about this. e.g. 'type r' will fetch the correct car
• User be aware: not all cars will have photos, and not all car types (transmission differences etc) will be represented by this tool
• For cars without a badge (Mazda3 2003 sedan has no badge), leave variant blank and no-badge will be assumed.
• if you do not know the variant, don't leave it blank. rather, write 'any' and the price range for all variants, from the cheapest to the most expensive, will be shown as one block.
• some cars have only one bodytype, in this instance, bodytype can be left blank. if left blank when there are multiple types, the first type listed on carsales URL directory will be assumed.
- > e.g. Audi TT 2013 has both coupe and convertible, and a no-badge and an 's' variant.
- > here, if variant is left blank, the no-badge version will be shown and if variant is 's', the tts model will be shown
- > if bodytype is specified, the correct version will be displayed. if blank, the first one (convertible in this instance) will be displayed.
